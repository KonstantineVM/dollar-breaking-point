Scope: this is a COVERAGE MEASUREMENT on the existing residence-resolved N-PORT panel,
using only the panel plus two committed lookups (GLEIF L2 map; filing-sourced ISIN
hand-map). It does NOT re-tag the operator, rebuild the operator, re-run separability, or

## THE PRE-REGISTERED DECISION RULE (stated before the numbers)

The threshold is applied to **B**, the DISCRIMINATING covered-value share =
value(assigned nationality ISO != residence ISO) / value(all haven). It is applied to **B,
NOT A**. Reason: A (coverage-any) counts GLEIF `SELF` rows whose assigned nationality
simply restates the issuer's own residence (e.g. a Cayman-resident LEI tagged KY). Those
rows are COSMETIC for the separability question — they carry no holder/issuer nationality
variation beyond what residence already encodes, so they cannot break the uniform aggregate
reweighting. Only the part of value where nationality != residence (hand-map CN on Cayman
ISINs; GLEIF real L2 parent to a third country; GLEIF SELF=CN on an HKG-resident H-share)
carries genuine per-security signal. That is B.

- **PROCEED branch:** if B (nationality != residence, by value) is HIGH (>~0.80) AND the
  uncovered tail is small or characterizable (untagged nationality plausibly bounded), then
  the RAS / max-entropy tail-fill is cosmetic and a re-tag would carry real holder-level
  nationality variation => RECOMMEND-PROCEED to a human-gated re-tag + separability re-test.
- **NOT-PROCEED branch:** if B is LOW, or the uncovered tail is large / Cayman-heavy /
  diffuse, then an entropy fill would DOMINATE the unmeasured cells and collapse to the
  aggregate reweighting (reproducing the ~+0.521 aggregate result) => RECOMMEND-NOT-PROCEED;
  the licensed CMNS crosswalk remains the only thing that closes identification.
- **OPEN:** if coverage is ambiguous or GLEIF was unusable.

### Diagnostic E — is A cosmetic? YES, majority.
58.1% of covered-any value (pooled) is the cosmetic b1 portion: GLEIF `SELF` rows whose
parent_country restates the issuer's own residence. So A=0.686 overstates usable coverage
by more than 2x relative to B=0.287. This is exactly the "A high but B low" case the rule
warns about: most of what looks covered is residence restated, which a re-tag would collapse
back onto the aggregate reweighting.

### D — Uncovered tail (31.4% of haven value): large, Cayman-heavy, diffuse.
- Residence split: **KY = 0.300** of total haven value (i.e. essentially the entire 0.314
  tail is Cayman), HKG = 0.0071, VGB = 0.0071. The uncovered mass is overwhelmingly Cayman.
- Concentration: issuer HHI = 0.0014; top-10 untagged issuers = only 6.7% of uncovered
  value. The tail is DIFFUSE, not a short list of bound-able names.
- Largest untagged issuers are Cayman structured-credit and hedge-fund vehicles, NOT
  Chinese operating companies: Madison Park Funding, Palmer Square Loan Funding, Carlyle
  Global Market Strategies, Dryden Senior Loan Fund, OZLM, Voya CLO, TICP CLO, Sound Point
  CLO, Vibrant CLO, CIFC Funding, Cerberus, Millennium / Elliott (hedge funds), plus
  Pagseguro Digital (Brazil-parent, not CN) and ANT INTERNATIONAL (small). No obvious
  large Chinese operating-company parent (Tencent/Meituan/Xiaomi etc.) sits unflagged at the
  top of the tail; the single "nio" flag is a false-positive substring inside "Dryden
  Senior Loan Fund" and is not a Chinese issuer. The tail's nationality is therefore NOT
  plausibly bounded to China — it is dominated by genuinely Cayman-domiciled CLO/fund
  vehicles whose nationality would have to be entropy-filled.

### Value-unit honesty (load-bearing).
`currency_value` is denominated in the per-row `currency_code`, NOT uniformly USD; the
provenance never confirmed USD denomination. On the positive haven subset value splits
~HKD 51% / USD 48% / minor ~1%, and currency correlates with residence (HKD concentrates in
HKG), so a raw mixed-currency sum would bias the coverage ratio. All sums are USD-EQUIVALENT:
USD x1, HKD /7.80 (HKMA Linked Exchange Rate System peg midpoint), minor currencies (<=1.4%)
at fixed representative rates. The B-sensitivity table above shows the ratio is invariant to
the minor-currency choice. This is a free-data coverage ratio, not an assertion that
`currency_value` is USD.

## WHICH BRANCH APPLIES: RECOMMEND-NOT-PROCEED

B = 0.287 pooled (max 0.360 in any quarter) is far BELOW the ~0.80 PROCEED threshold, and B
falls over the panel (0.36 in 2019Q4 -> 0.18 in 2024Q4). Simultaneously the uncovered tail
is LARGE (31.4%), Cayman-heavy (KY = 0.300 of total haven), and DIFFUSE (HHI 0.0014, top-10
= 6.7%), with no bound-able Chinese-parent concentration. Both failing conditions of the
NOT-PROCEED branch are met; neither PROCEED condition holds.

Therefore: **RECOMMEND-NOT-PROCEED.** A re-tag built on this free map would leave ~31% of
haven value (almost entirely diffuse Cayman vehicles) to a RAS / max-entropy tail-fill, and
with B only ~0.29 the informative per-security variation is too thin to dominate that fill.
The entropy fill would dominate and the re-tagged cell would collapse toward the aggregate
reweighting (reproducing the ~+0.521 aggregate result), not provide independent holder-level
identification. The licensed CMNS ultimate-parent crosswalk remains the only thing that
would close identification.

This recommendation is ADVISORY. Whether the re-test runs is the human's call; this artifact
neither re-tags nor re-tests anything.

### Symmetric statement (both directions).
For completeness and against planting the verdict: the hand-map + GLEIF DO surface a real,
non-trivial per-security China signal — C shows CN-parent = 22.6% of total haven value
pooled (11.5%–31.1% across quarters), and B's informative component is genuine (hand-map CN
on Cayman ISINs and GLEIF SELF=CN on HKG H-shares). That signal is not zero and is not
cosmetic. The reason it still does not clear the bar is quantitative, not definitional:
0.287 < 0.80 and a 31% diffuse Cayman tail would be entropy-filled, not measured. Had B come
in above ~0.80 with a small/characterizable tail, the same rule and same script would have
returned RECOMMEND-PROCEED. The branch is selected by the measured numbers, not assumed.
