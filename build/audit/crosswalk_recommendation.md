# Construct-the-Crosswalk Coverage — RECOMMENDATION (ADVISORY)

Scope: a COVERAGE MEASUREMENT. It constructs a free, official per-security China-nationality
crosswalk (SEC HFCAA + EDGAR F-6/20-F + VIE-look-through Inline-XBRL OpCo jurisdiction + GLEIF
ISIN→LEI / QCC / OpenCorporates glue), applies it to the residence-resolved N-PORT haven panel, and
RE-MEASURES discriminating coverage **B** for direct comparison to the prior pass's 0.287. It does NOT
re-tag the operator, rebuild it, re-run separability, or touch DP2–DP6. Every number is regenerated by
`build/audit/crosswalk_coverage_recompute.py` from the tagged panel + the two lookups alone (no network);
`build/audit/crosswalk_coverage.json` is byte-identical to a fresh re-run (orchestrator-executed). No date,
no probability, no hazard claim.

---

## THE PRE-REGISTERED DECISION RULE (stated before the numbers)

The threshold is applied to **B** = discriminating covered value share = value(constructed
parent-nationality ≠ residence) / value(all haven). This is the only part of coverage that carries
holder/issuer nationality variation BEYOND residence — the part a re-tag would need to break the uniform
aggregate reweighting that produced the prior ~+0.52 separability sign.

- **RECOMMEND-PROCEED** iff **B clears ~0.80** by value with a bounded/characterized uncovered tail →
  a human-gated re-tag + separability re-test becomes worth running.
- **RECOMMEND-NOT-PROCEED** if **B remains well short of 0.80** → state how far the construction moved B
  from 0.287 and what residual value remains; the licensed CMNS crosswalk stays the only closer for that
  residual.
- **OPEN** if sources were unfetchable or coverage is ambiguous.

## WHICH BRANCH APPLIES: **RECOMMEND-NOT-PROCEED**

**B = 0.3425 is far below the ~0.80 PROCEED threshold**, and the uncovered tail is large (0.310),
Cayman-heavy (KY 0.296), and diffuse (HHI 0.0014; top-10 6.8%) with no bound-able Chinese-parent
concentration. Both failing conditions of the NOT-PROCEED branch hold; neither PROCEED condition holds.

Therefore: **RECOMMEND-NOT-PROCEED.** The free SEC-HFCAA + XBRL-VIE + GLEIF/QCC construction adds **real,
correctly-sourced** per-security China nationality (it moves B from 0.287 to 0.3425 and now tags every
SEC-listed Chinese VIE/ADR in the panel, NetEase included), but ~31% of haven value — almost entirely
diffuse, genuinely non-Chinese Cayman structured credit — remains without a non-residence nationality. A
re-tag built on this crosswalk would still hand that 31% to a RAS/max-entropy fill that would dominate and
collapse back toward the aggregate reweighting (the ~+0.52 result the FWTW reconstruction confirmed neither
max-ent nor RAS can escape). The **licensed CMNS ultimate-parent crosswalk remains the only closer** for the
residual.
